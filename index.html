  • Authentication

    Authentication for the Yungas Api

    • Authentication

      no implementation for this version
      Given the user e-mail, password and apikey, returns a Bearer token valid for the next 5 minutes.
      Request:
      Resource:
      https://app.yungas.com.br/api/authentication/
      Body:
      {
      "apikey": integer, Number of the month that this revenue belongs (1-12)
      "email": integer, ID of the franchise that you will add the revenue
      "password": integer Number of the year that this revenue belongs (YYYY)
      }
      Response:
      Body:
      string Bearer token valid for the next 5 minutes, if expires you must authenticate again.
      Codes:
      Code Message Description
      401 Authentication failed You must insert Bearer token on header Authentication
      Sample:
      Request:
      POST authentication/ HTTP/1.1 Content-Type: application/json; charset=utf-8 { "apikey": 42, "email": 42, "password": 42 }
      Response:
      HTTP/1.1 200 OK Content-Type: application/json; charset=utf-8 "my_response"

  • Finances

    Input revenue information for a specific franchise

    • RevenueAdd

      no implementation for this version
      Input revenue infos for a specific franchise on a specific month and year
      Request:
      Resource:
      https://app.yungas.com.br/api/finances/
      Body:
      {
      "franchise": integer, ID of the franchise that you will add the revenue
      "month": integer, Number of the month that this revenue belongs (1-12)
      "planned": number, Revenue expected for the following period | (10000.00) as Ten thousand.
      "revenue": number, Revenue value for the following period | (10000.00) as Ten thousand.
      "year": integer Number of the year that this revenue belongs (YYYY)
      }
      Response:
      Body:
      {
      "created_by": integer, ID of the user that create the revenue
      "franchise": integer, ID of the franchise that you will add the revenue
      "franchisor": integer, ID of the franchisor
      "id": integer, ID of revenue that you just created
      "month": integer, Number of the month that this revenue belongs (1-12)
      "planned_value": number, Revenue expected for the following period | (10000.00) as Ten thousand.
      "revenue_value": number, Revenue value for the following period | (10000.00) as Ten thousand.
      "year": integer Number of the year that this revenue belongs (YYYY)
      }
      Codes:
      Code Message Description
      401 Authentication failed You must insert Bearer token on header Authentication
      404 Franchise not found Franchise does not exists
      422 Missing parameter Missing one or more parameters
      Sample:
      Request:
      POST finances/ HTTP/1.1 Content-Type: application/json; charset=utf-8 { "franchise": 42, "month": 42, "planned": 13.37, "revenue": 13.37, "year": 42 }
      Response:
      HTTP/1.1 200 OK Content-Type: application/json; charset=utf-8 { "created_by": 42, "franchise": 42, "franchisor": 42, "id": 42, "month": 42, "planned_value": 13.37, "revenue_value": 13.37, "year": 42 }

  • Franchise

    Returns data from a specific franchise

    • Get

      no implementation for this version
      Returns the data from a specific franchise
      Request:
      Resource:
      https://app.yungas.com.br/api//franchise/?id={franchise_id}
      URI Parameters:
      Parameter Type Optional Description
      franchise_id integer Franchise id, you can obtain this id consulting on FranchisorDetail method
      Response:
      Body:
      {
      "Franchise": Franchise
      }
      Codes:
      Code Message Description
      401 Authentication failed You must insert Bearer token on headers
      404 Franchise not found Franchise does not exists
      422 Missing parameter You must insert the franchise_id on the get url
      Sample:
      Request:
      GET /franchise/?id=42 HTTP/1.1 Content-Type: application/json; charset=utf-8
      Response:
      HTTP/1.1 200 OK Content-Type: application/json; charset=utf-8 { "Franchise": { "city": "my_city", "created_on": "my_created_on", "id": 42, "name": "my_name" } }

  • Franchisor

    Return the Franchisor info

    • Get

      no implementation for this version
      Returns the franchisor data, inclusing all his franchises
      Request:
      Resource:
      https://app.yungas.com.br/api//franchisor/
      Response:
      Body:
      {
      "Franchisor": [
      {
      "id": integer, Object identifier
      "name": string Name of the object
      }
      ]
      }
      Codes:
      Code Message Description
      401 Authentication failed You must insert Bearer token on headers
      404 Franchise not found Franchise does not exists
      422 Missing parameter You must insert the franchise_id on the get url

      Request:
      POST /franchisor/ HTTP/1.1 Content-Type: application/json; charset=utf-8

      HTTP/1.1 200 OK Content-Type: application/json; charset=utf-8 { "Franchisor": [ { "id": 42, "name": "my_name" }, { "id": 42, "name": "my_name" } ] }
